# Step 2: Research Context

Gather information needed to decompose the goal intelligently.

## Process

1. **Read any referenced materials:**
   - Backlog items identified in step 1
   - Template repos or specs the user mentioned
   - Existing code or docs that relate to the goal

2. **Web research if needed:**
   - Only if the goal involves external tools, frameworks, or APIs
   - Capture key findings that affect task structure
   - Don't over-research — just enough to decompose

3. **Identify the pattern:**
   - Is this a BUILD task? (create something new)
   - Is this a REFACTOR task? (change existing things)
   - Is this a RESEARCH task? (produce documentation)
   - Is this a TEST task? (validate existing work)

4. **Map dependencies:**
   - What must exist before the first task can start?
   - Which tasks depend on other tasks' output?
   - Are there external dependencies (APIs, accounts, human decisions)?

## Output

Internal — no report. Findings feed directly into step 3.

## Rules

- Spend at most 2-3 actions on research
- If the goal is straightforward, skip research entirely
- Never research as a way to avoid starting work
